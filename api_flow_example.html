🚀 Nadia API - Complete Purchase Flow

This page demonstrates the complete flow for purchasing XL packages using the Nadia API.

📋 Complete API Flow

Step 1: Request OTP

First, request an OTP for the phone number that will receive the package.

POST /api/otp/request Content-Type: application/json { "phone_number": "087786388052" }
{ "status_code": 200, "message": "OTP sent successfully", "success": true, "data": { "phone_number": "087786388052", "can_resend_in": 60, "expires_in": 300 } }

Step 2: Verify OTP & Get Access Token

Verify the OTP code received via SMS to get the access token.

POST /api/otp/verify Content-Type: application/json { "phone_number": "087786388052", "otp_code": "123456" }
{ "status_code": 200, "message": "OTP verified successfully", "success": true, "data": { "access_token": "1097690:ec321a89-6593-4b01-9a9e-383744301283", "phone_number": "087786388052" } }

Step 3: Search Available Packages (Optional)

Search for available packages to find the right package code.

POST /api/packages/search Content-Type: application/json { "query": "30 hari", "category": "data" }

Step 4: Purchase Package

Purchase the package using the access token from Step 2.

POST /api/purchase Content-Type: application/json { "phone_number": "087786388052", "package_code": "XL_MASTIF_30D_P_V1", "payment_method": "BALANCE", "access_token": "1097690:ec321a89-6593-4b01-9a9e-383744301283", "source": "telegram_bot" }
{ "status_code": 200, "message": "Package purchased successfully", "success": true, "data": { "trx_id": "TRX123456789", "package_name": "XL Mastif 30 Hari", "amount": 50000, "package_processing_fee": 1000 } }

📊 Source Tracking

The API supports source tracking to monitor where transactions come from:

  • telegram_bot - Transactions from Telegram Bot
  • whatsapp_bot - Transactions from WhatsApp Bot
  • web_interface - Transactions from Web Interface
  • api_direct - Direct API calls (default)

This helps you track performance and success rates across different channels in the dashboard.
